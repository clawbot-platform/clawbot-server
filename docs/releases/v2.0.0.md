# clawbot-server v2.0.0 Release Notes

## Release scope

`clawbot-server` v2.0.0 is the first semver-tagged release for the validated governed execution stack used by ACH week-run workflows.

Included and validated in this release:

- run/cycle lifecycle APIs under `/api/v1`
- governed execution rings (`ring_0` to `ring_3`)
- reviewer actions (`approve`, `reject`, `override`, `defer`)
- deterministic, llm, and dual execution modes
- dual comparison persistence and retrieval
- governance audit hash-chaining
- model/guardrail profile support used by ACH week-run integration

## Images

- image: `ghcr.io/clawbot-platform/clawbot-server`

## Install and run from released image

1. Prepare core dependencies (Postgres, Redis, NATS):

```bash
cp .env.example .env
docker compose --env-file .env \
  -f deploy/compose/docker-compose.yml \
  -f deploy/compose/docker-compose.override.yml \
  up -d postgres redis nats
```

2. Run the released control-plane image:

```bash
docker run -d --name clawbot-server \
  --network clawbot-foundation_foundation \
  -p 127.0.0.1:8080:8080 \
  -e APP_ENV=production \
  -e SERVER_ADDRESS=0.0.0.0:8080 \
  -e AUTO_MIGRATE=true \
  -e DATABASE_URL='postgres://clawbot:REQUIRED_SECRET_replace_me_postgres@postgres:5432/clawbot?sslmode=disable' \
  -e REDIS_URL='redis://redis:6379' \
  -e NATS_URL='nats://nats:4222' \
  -e INFERENCE_BASE_URL='http://ai-precision:11434' \
  ghcr.io/clawbot-platform/clawbot-server:v2.0.0
```

3. Smoke checks:

```bash
curl -s http://127.0.0.1:8080/healthz | jq
curl -s http://127.0.0.1:8080/readyz | jq
curl -s http://127.0.0.1:8080/version | jq
curl -s http://127.0.0.1:8080/api/v1/ops/overview | jq
```
